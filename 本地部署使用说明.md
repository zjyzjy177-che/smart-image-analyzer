# 智能图片分析系统：完整本地部署使用说明

本项目使用 Git LFS 保存约 1.44 GB 的离线模型。必须使用 **Git 克隆 + Git
LFS 拉取**，不要直接下载 GitHub 的 Source code ZIP；ZIP 或未安装 LFS 的克隆
可能只得到约 130 字节的模型指针。

## 1. 新电脑首次安装

### Windows

1. 安装 Git、Git LFS 和 Python 3.10～3.13。
2. 在 PowerShell 执行：

```powershell
git lfs install
git clone https://github.com/zjyzjy177-che/smart-image-analyzer.git
cd smart-image-analyzer
git lfs pull
py -3 -m pip install -r requirements.txt
.\run.bat
```

浏览器访问 <http://localhost:7860>。双击 `run.bat` 也可以启动；脚本会检查并
安装缺失的 Python 依赖，但 Git LFS 模型必须先拉取完整。

### Linux/macOS

```bash
git lfs install
git clone https://github.com/zjyzjy177-che/smart-image-analyzer.git
cd smart-image-analyzer
git lfs pull
python3 -m pip install -r requirements.txt
bash run.sh
```

## 2. 已有仓库更新

```bash
git status
git pull --ff-only origin main
git lfs pull
```

如果 `git pull` 提示本地文件会被覆盖，先不要执行 `reset --hard` 或删除模型；
确认本地改动归属后再处理。

## 3. 检查模型是否完整

```powershell
git lfs ls-files
Get-Item models\resnet50\resnet50-11ad3fa6.pth
Get-Item models\blip-image-captioning-base\pytorch_model.bin
Get-Item models\opus-mt-en-zh\pytorch_model.bin
Get-Item models\easyocr\craft_mlt_25k.pth
Get-Item models\easyocr\zh_sim_g2.pth
```

正常大小约为：

| 模型 | 大小 |
|---|---:|
| ResNet50 | 98 MB |
| BLIP | 945 MB |
| MarianMT 英译中 | 301 MB |
| EasyOCR 检测模型 | 79 MB |
| EasyOCR 中英文识别模型 | 21 MB |

如果文件只有约 130 字节，或开头是
`version https://git-lfs.github.com/spec/v1`，它是正常的 LFS 指针，但尚未被
展开。执行：

```bash
git lfs install
git lfs pull
git lfs checkout
```

不要删除指针，不要把 `torch.load(..., weights_only=True)` 改成 `False`。

## 4. 为什么之前组长环境报错

`Unsupported operand 118` 中的 118 是字符 `v` 的编码。PyTorch 实际读取到的
不是模型，而是以 `version ...` 开头的 Git LFS 指针。

后续错误属于连锁反应：

- `corrupt_msg is not defined`：EasyOCR 尝试把 LFS 指针解析成权重。
- `'NoneType' object is not callable`：首次模型加载失败后使用了半初始化缓存。

当前代码会主动识别 LFS 指针并给出拉取命令，而且只有模型完整加载后才写入全局
缓存。

## 5. 本机能运行不代表部署包完整

Hugging Face、Torch 和 EasyOCR 默认会把模型保存在用户目录。开发者电脑可能
因为已有缓存而正常运行，即使项目内模型已经缺失。检查交付完整性时必须：

1. 检查 `models/` 中的真实文件大小。
2. 设置 `HF_HUB_OFFLINE=1`、`TRANSFORMERS_OFFLINE=1` 后测试。
3. 不得用“我的电脑可以运行”代替模型文件检查。

## 6. 常见问题

- **提示 Python 未安装：** Windows 优先使用 `py -3 --version` 检查启动器。
- **首次打开较慢：** BLIP、翻译和 OCR 首次加载需要几十秒。
- **出现 LFS 配额或下载错误：** 确认 GitHub 登录状态和仓库 LFS 配额。
- **只需要运行，不需要 Git 历史：** 完整执行 `git lfs pull` 后，可以复制项目
  工作目录作为离线交付包；不要复制 `.git` 目录。
